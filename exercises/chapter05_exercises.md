# 第5章演習問題：防御戦術 - サイトホールドとリテイク

## 🎯 演習の目的

この演習では、VALORANTにおける防御戦術の基本概念、アンカリング戦術、リテイク戦術の実行判断について理解度を確認します。

## 📝 演習問題

### 問題1：防御戦術の3段階（基礎）

Haven防御側でラウンド開始から25秒が経過。以下の情報が得られました。現在どの段階にいるべきか、次に取るべき行動を選んでください。

**現在の状況:**
- A Long: 足音なし
- B Main: 1人確認
- C Long: 2人確認、ユーティリティ使用中
- 残り時間: 40秒

- A) Information Gathering段階 - さらなる情報収集を継続
- B) Delay & Hold段階 - C Longでアンカリング強化
- C) Decision & Retake段階 - 早期ローテーション実行
- D) 全員Cサイト集合で数的優位作成

<details>
<summary>正解と解説を見る</summary>

**正解: B) Delay & Hold段階 - C Longでアンカリング強化**

**解説:**
防御戦術の3段階の時間配分と現在の状況分析：

- **Information Gathering（0-30秒）**: ほぼ完了
- **Delay & Hold（30-45秒）**: 現在の段階（25秒経過、40秒残り）
- **Decision & Retake（45秒-終了）**: これから準備する段階

**25秒時点での最適行動（40秒残り）:**
- C Longの2人確認 = メイン攻撃の可能性高
- ユーティリティ使用中 = 本格的プッシュ開始の兆候
- B Mainの1人 = 牽制またはフェイクの可能性
- A Long音なし = 現時点では脅威低

**B選択の理由:**
- C Longアンカーを強化してプッシュを遅延
- 情報収集は十分、今は遅延に集中すべき
- 40秒残りは適切な遅延タイミング
- B、A警戒は継続しつつCに重点配置

早期ローテーション（C選択）は情報不足で危険、全員集合（D選択）は他サイト無防備で不適切です。
</details>

### 問題2：アンカリング戦術の実行判断（応用）

Bind A Site防御でCypherがアンカリング中です。以下の状況でどの行動を取るべきでしょうか？

**状況:**
- 敵チーム: A Shortから3人プッシュ開始
- 自身: Cypher、体力75、ワイヤー・ケージ未使用
- チーム状況: B Siteに2人、Midに1人、ローテーション可能
- 残り時間: 30秒

- A) A Siteで最後まで抵抗、ワイヤーとケージで最大限遅延
- B) トラップをセットしてA Bathに戦術撤退、リテイク準備
- C) 即座にローテーション要求、3vs3の数的均衡を作る
- D) A ShortでOffアングルから1キル狙い、その後状況判断

<details>
<summary>正解と解説を見る</summary>

**正解: D) A ShortでOffアングルから1キル狙い、その後状況判断**

**解説:**
効果的なアンカリングの4要素を考慮した最適解：

**現在の状況分析:**
- **敵の脅威**: 3人プッシュ = 高い脅威レベル
- **自身の状況**: 体力75、ユーティリティ充実 = 戦闘継続可能
- **チーム状況**: ローテーション可能 = 支援期待できる
- **時間**: 30秒 = アンカリングに適切な時間

**選択Dの戦術的合理性:**

**1. ポジション選択（Offアングル）:**
- 敵の予想外位置で初撃優位
- A Shortの複数の退路確保
- 情報収集継続可能

**2. ユーティリティ管理（段階的使用）:**
- 初撃後、状況に応じてワイヤー・ケージ使用
- リテイク時のリソース温存可能
- 最大効果のタイミング調整

**3. 時間管理（適切な保持時間）:**
- 1キル狙いで敵の勢い削ぐ
- 成功なら継続、失敗なら撤退の柔軟性
- チームローテーション時間稼ぎ

**4. 心理戦（プレッシャー与える）:**
- 敵の突撃テンポ破綻
- 予想外の抵抗で混乱誘発

**他の選択肢の問題:**
- A) 最後まで抵抗: 3vs1は不利すぎて無駄死リスク
- B) 即座撤退: 敵に簡単にサイト放棄で時間稼ぎ不足
- C) 即座ローテーション: アンカー放棄で敵のフリープッシュ許可

効果的なアンカリングは無謀な抵抗ではなく、計算された遅延戦術です。
</details>

### 問題3：リテイク戦術の実行タイミング（実践）

Haven A Siteにスパイク設置完了、敵3人生存、味方4人でリテイクを計画中です。最適なリテイクタイミングはどれでしょうか？

**状況詳細:**
- スパイク位置: A Site中央
- 敵配置: 2人A Site、1人A Long
- 味方構成: Jett、Sova、Omen、Breach（全員フルヘルス）
- 残り時間: 35秒（スパイク設置から10秒経過）

- A) 即座にQuick Retake実行、敵のセットアップ前に突入
- B) 15秒待ってLate Retake、敵ユーティリティ消費後に実行
- C) 25秒でStall & Defuse、時間ギリギリで解除狙い
- D) ユーティリティで牽制のみ、次ラウンドの経済保護を優先

<details>
<summary>正解と解説を見る</summary>

**正解: B) 15秒待ってLate Retake、敵ユーティリティ消費後に実行**

**解説:**
リテイク成功の5つの要件に基づいた戦術的判断：

**状況分析:**
- **スパイク時間**: 45秒中35秒残り = 十分な時間
- **敵配置**: 2+1の分散配置 = 各個撃破の可能性
- **味方構成**: バランス良い攻撃構成
- **経過時間**: 設置から10秒 = 敵セットアップ進行中

**Late Retakeの戦術的優位:**

**1. 情報の正確性向上:**
- 15秒待つ間にSovaドローンで詳細偵察
- 敵の最終ポジション確認
- ユーティリティ残量把握

**2. 適切なタイミング:**
- 敵の初期ユーティリティ（スモーク等）が消費される
- セットアップ完了後の油断を狙える
- 25秒残しで十分な実行時間確保

**3. 役割分担の最適化:**
- Jett: A Long牽制でフランク阻止
- Sova: 情報収集とA Site支援
- Omen: A Siteスモークで安全突入
- Breach: フラッシュで敵の視界妨害

**4. ユーティリティ連携:**
- Omenenスモーク + Breachフラッシュの完璧な組み合わせ
- Sovaダーツで敵位置把握
- 段階的ユーティリティ使用で効果最大化

**5. 実行の統一性:**
- 15秒の準備時間で完璧な同期可能
- 各自の役割明確化
- 状況変化への対応準備

**他の選択肢の問題:**
- A) Quick Retake: 敵セットアップ済みで不利
- C) Stall & Defuse: 35秒残りでリスクが高すぎる
- D) 経済保護: 4vs3の有利な状況を無駄にする

**成功の鍵:**
- Sovaの情報収集（10-15秒）
- 全員の準備完了確認（15-20秒）
- 統一タイミングでの実行（20-25秒）
- 残り15秒で確実な制圧とスパイク解除

効果的なリテイクは焦らず、準備を整えてから実行することが重要です。
</details>

### 問題4：防御ポジショニングの理解（高度）

Split B Site防御でSageがポジション取りをします。以下の4つのポジションのうち、最も効果的な防御位置はどれでしょうか？

**マップ状況:**
- B Main方向から敵のプッシュが予想される
- チーム: B Site（Sage）、Mid（Omen）、A Site（Raze、Cypher）
- 経済: フルバイラウンド
- 残り時間: ラウンド開始時点

**ポジション選択肢:**
- A) B Mainの角で直接対峙、ウォールで通路封鎖
- B) B Site中央のBoxes後方、Slow Orbで遅延
- C) B Site後方のOff-Angle、ヒール準備完了
- D) B Heaven高台、上からの射角確保

<details>
<summary>正解と解説を見る</summary>

**正解: C) B Site後方のOff-Angle、ヒール準備完了**

**解説:**
効果的な防御ポジショニングの原則に基づいた最適解：

**ポジショニングの3要素分析:**

**1. 角度優位の確保:**
- Off-Angleは敵の予想外位置
- ピーカーズアドバンテージを無効化
- 初撃優位の確保可能

**2. 生存性の重視:**
- B Site後方は複数の退路確保
- Mid、A Siteへの迅速ローテーション可能
- 危険時の即座撤退オプション

**3. チーム貢献の最大化:**
- ヒール準備でチーム支援可能
- 情報伝達の安全な位置
- リテイク時の適切なポジション

**選択Cの戦術的優位性:**

**地形活用:**
- Splitの狭いB Siteで意外性のあるポジション
- 敵の定型的なクリアリングを回避
- サイト全体の視野確保

**ユーティリティ最適化:**
- Slow Orb: B Mainチョークポイントに事前設置
- Barrier Orb: 状況に応じてB MainまたはSite内封鎖
- Healing Orb: 味方支援またはセルフヒール

**チーム連携:**
- Omen（Mid）との連携容易
- A Siteチームとの情報共有円滑
- 全体戦術への柔軟な対応

**経済効率:**
- フルバイラウンドでの確実性重視
- 無駄死回避で次ラウンド経済確保
- チーム全体の経済安定性寄与

**他の選択肢の問題:**
- A) B Main直接対峙: ピーカーズアドバンテージで不利、退路なし
- B) Boxes後方: 定型的位置で敵の予想範囲内
- D) B Heaven: Splitの構造上、孤立しやすく支援困難

**実戦での応用:**
- 敵プッシュ開始時: Slow Orbで遅延、情報伝達
- 交戦時: Off-Angleから有利な撃ち合い
- 劣勢時: 安全な撤退でリテイク準備
- 勝勢時: 前進してサイト完全制圧

効果的な防御ポジションは、単独での戦闘力よりもチーム戦術への貢献度を重視します。
</details>

### 問題5：情報伝達とチーム連携（総合）

Lotus防御側で複数方向から同時攻撃を受けています。最も効果的なコミュニケーションと対応はどれでしょうか？

**状況:**
- A Main: 2人プッシュ確認
- B Main: 1人確認、音継続
- C Main: 3人確認、スモーク展開済み
- 自チーム配置: A（Killjoy）、B（Fade）、C（Viper、Omen）、Lurk（Yoru）

**対応選択肢:**

- A) 「C Long 3人 フルバイ スモーク有り 即ローテーション求む」
- B) 「敵多い 助けて」
- C) 「C確定 A、B放棄 全員Cに集合」
- D) 「C 3人確認 メイン攻撃 A 2人 B 1人 各自判断」

<details>
<summary>正解と解説を見る</summary>

**正解: A) 「C Long 3人 フルバイ スモーク有り 即ローテーション求む」**

**解説:**
効果的な情報伝達フォーマットと戦術判断の模範解答：

**情報伝達フォーマットの要素:**
```
「[場所] [敵の人数] [装備] [行動] [時間]」
「C Long 3人 フルバイ スモーク有り 即ローテーション求む」
```

**選択Aの優秀な点:**

**1. 完全な情報提供:**
- **場所**: C Long（具体的位置）
- **人数**: 3人（数的脅威レベル明確）
- **装備**: フルバイ（経済状況把握可能）
- **行動**: スモーク有り（戦術的準備完了）
- **要求**: 即ローテーション（明確な行動指示）

**2. 戦術的判断の妥当性:**
- C Main 3人 = 明確なメイン攻撃
- フルバイ + スモーク = 本格的エクスキュート
- A 2人、B 1人 = 牽制・フェイクの可能性高
- 数的優位確保が勝利への最短路

**3. 緊急性の適切な伝達:**
- 「即ローテーション」= 時間的切迫感
- 明確な行動指示でチーム判断速度向上
- 混乱防止の簡潔かつ完全な情報

**Lotus 3サイト防御の戦術的考察:**

**ローテーション判断基準:**
- C 3人確認 = 60%の敵戦力集中
- スモーク展開 = エクスキュート開始の明確な兆候
- A、B軽攻撃 = 分散戦術の可能性低い

**最適配置変更:**
```
変更前: A(1), B(1), C(2), Lurk(1)
変更後: A(1軽防御), B(0), C(3重防御), Support(1)
```

**他の選択肢の問題:**
- B) 情報不足で判断不可能、パニック的で非建設的
- C) 極端すぎる判断、A完全放棄のリスク大
- D) 判断を個人に委ねる無責任な伝達、チーム連携阻害

**実戦での応用例:**
```
良い例:
「A Long 2人 Vandal中心 プッシュ開始 35秒」
「B落ちた リテイク準備」

悪い例:
「敵いる」
「死んだ」
「やばい」
```

**効果的コミュニケーションの要素:**
1. **具体性**: 正確な数値と位置
2. **簡潔性**: 必要十分な情報のみ
3. **迅速性**: 判断に必要なタイミング
4. **建設性**: 次の行動への明確な指針

優れた情報伝達は、単なる報告ではなく戦術的判断を支援するツールです。
</details>

### 問題6：経済状況に応じた防御戦術（実戦応用）

チームの経済状況が悪化し、次ラウンドがエコラウンドになります。Haven防御側でどの戦術を選択すべきでしょうか？

**経済状況:**
- 自チーム: 平均2000クレジット、フルバイ不可
- 敵チーム: フルバイ確定、高性能装備
- 前ラウンド: 敵のA攻撃で敗北
- マップ: Haven（3サイト）

**戦術選択肢:**

- A) 3サイト通常配置でエコ装備による堅実な防御
- B) A Site集中配置（4人）、他サイト放棄のスタック戦術  
- C) 全員でA Longプッシュ、敵陣深くでのピック狙い
- D) バラバラ配置でルーク、個人技による混乱戦術

<details>
<summary>正解と解説を見る</summary>

**正解: B) A Site集中配置（4人）、他サイト放棄のスタック戦術**

**解説:**
経済劣勢時の防御戦術における最適解：

**エコラウンド防御の基本原則:**
1. **装備差の現実受容**: 正面衝突は不利
2. **数的優位の創出**: 局所的な有利状況作成
3. **予測とリスクテイク**: 大胆な戦術変更
4. **経済効率最大化**: 最小投資で最大効果

**選択Bの戦術的合理性:**

**1. 前ラウンド情報の活用:**
- 前ラウンドA攻撃 = 敵のA攻撃パターン把握済み
- 同じ戦術継続の可能性60-70%
- 敵の成功体験による再実行期待値高

**2. 数的優位の戦術的価値:**
```
通常配置: 各サイト1-2人 vs 敵3-4人攻撃 = 劣勢
スタック配置: A Site 4人 vs 敵3-4人攻撃 = 均衡〜優勢
```

**3. エコ装備の効率最大化:**
- 4人の集中火力で装備差相殺
- Sheriffヘッドショットの確率向上
- アーマー購入の集中投資効果

**4. 心理的効果:**
- 敵の予想外配置で混乱誘発
- 強気な防御で敵の慎重化
- チームの士気向上効果

**具体的実行例（Haven A Site）:**
```
配置:
- A Long: 2人（Cypher、Sova）
- A Short: 1人（Raze）  
- A Site: 1人（Omen）
- B、C: 完全放棄

戦術:
- 情報収集で敵A攻撃確認
- Cypherトラップ + Sovaダーツで早期発見
- 4人集中火力で装備差克服
- 成功時、敵経済にダメージ
```

**リスク管理:**
- B、C攻撃の場合は諦めて次ラウンド準備
- 予測外れリスク40%は経済劣勢時の必要リスク
- 成功時の経済回復効果が失敗リスクを上回る

**他の選択肢の問題:**
- A) 通常配置: 装備差で各サイト確実に突破される
- C) A Longプッシュ: 敵陣深くでの不利な撃ち合い
- D) バラバラ配置: 個人技頼みで安定性なし

**エコラウンド成功の3要素:**
1. **大胆な予測**: 敵の行動パターン分析
2. **集中投資**: リソースの一点集中
3. **チーム連携**: 4人の完璧な連携実行

**長期的戦略価値:**
- 1ラウンド取得で経済回復
- 敵の心理的プレッシャー増大
- 次ラウンドのフルバイ可能性

エコラウンドは守備的になりがちですが、時には攻撃的リスクテイクが必要です。
</details>
